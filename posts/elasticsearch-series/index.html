creating a small app with ES as backend #

This is a series about a slack related app where we use ES as the backend for every event we would possibly like to store for analysis.

There are few things we need to consider and the first is related to the fields provided by the slack api that are related to dates and timestamps.

Most of them are ending with ts. By storing all events in ES we can then go and identify them.

mappings #

The following will help construct mapping for the date fields:

mappings = {"properties":{}}
date_field = {
      "type": "date",
      "format": "strict_date_optional_time||epoch_second||epoch_millis"
    }
for k in date_fields.split():
    mappings['properties'][k]=date_field

print(json.dumps(mappings, indent=2))

reindexing #

Whenever we do a mapping change we should reindex data from the source index to the one we changed the mapping in order for the data to be parsed according to the type of the filed we changed (in the destination index)

POST _reindex 
    {
      "source": {
        "index": "slack-messages"
      },
      "dest": {
        "index": "slack-messages_reindexed"
      }}

aliasing #

It is expected to have mapping changes as you enhance or develop an app thus it is often recommended to get an alias in place to use when sending data. That will also provide the ability to perform changes without having to affect application and to minimise downtime

Have a look at mapping with 0 downtime to get an idea

kibana #

So we have our data in es and we want to search them. Most of what we might want to do is already available.. For kibana to show the date histogram of our documents the timestamp field (here we marked ts field of slack messages for that) to be the one used for this purpose.

formating #

Some slack messages or events contain urls in order to make them clickable in kibana you need to change the format of those fields so that are presented as links. Navigate to stack management (under kibana)index management <your index patterm> and find your field to do an edit. While in the “edit” of that field, enable set format and switch to Url.
